# 博客系统需求分析文档

## 一、项目概述

### 1.1 项目背景
本项目旨在开发一个功能完善的多用户博客平台，支持文章发布、社交互动、AI辅助写作等功能。系统采用前后端分离架构，为用户提供现代化的内容创作和分享体验。

### 1.2 项目目标
- 提供稳定可靠的博客内容管理平台
- 支持多用户协作和社交互动功能
- 集成AI技术辅助内容创作
- 实现高性能的内容查询和推送机制
- 保障系统安全性和数据完整性

### 1.3 目标用户
- **内容创作者**: 发布和管理个人博客文章
- **普通读者**: 浏览、评论、收藏感兴趣的内容
- **系统管理员**: 管理用户、分类和系统数据

## 二、功能需求

### 2.1 用户管理模块

#### 2.1.1 用户注册
- **需求描述**: 新用户通过用户名和密码注册账号
- **输入要求**:
  - 用户名: 3-20字符，支持字母、数字、下划线、中文
  - 密码: 6-20字符
- **业务规则**:
  - 用户名唯一性校验
  - 密码使用BCrypt加密存储
  - 使用雪花算法生成用户ID

#### 2.1.2 用户登录
- **需求描述**: 用户通过用户名和密码登录系统
- **功能要点**:
  - BCrypt密码验证
  - 生成JWT访问令牌（有效期2小时）
  - 生成刷新令牌（有效期7天）
  - 刷新令牌存储在Redis中
- **安全机制**:
  - 支持令牌撤销（登出时删除Redis中的令牌）
  - 令牌黑名单机制

#### 2.1.3 用户信息管理
- **查询用户信息**: 支持按用户名查询单个用户或分页查询用户列表
- **更新用户信息**: 用户可修改用户名和密码，只能修改本人信息
- **删除用户**: 支持用户注销账号（需管理员权限或本人操作）
- **用户数据汇总**: 一次性获取用户的所有文章、点赞、收藏、评论数据

### 2.2 文章管理模块

#### 2.2.1 文章发布
- **需求描述**: 用户创建和发布博客文章
- **输入要求**:
  - 标题: 1-200字符，必填
  - 内容: 1-50000字符，必填
  - 分类: 必须选择已存在的分类
  - 状态: 草稿(0)、已发布(1)、已下架(2)
- **业务规则**:
  - 自动设置作者ID为当前登录用户
  - 敏感词过滤（使用DFA算法）
  - 自动记录创建时间和更新时间
  - 发布时通过WebSocket通知所有粉丝
  - 文章ID添加到布隆过滤器

#### 2.2.2 文章查询
- **单篇文章查询**:
  - 通过文章ID查询详情
  - 布隆过滤器预判断文章是否存在
  - Redis缓存机制（5分钟过期）
  - 返回文章完整信息（标题、内容、作者、时间、点赞数、收藏数）

- **文章列表查询**:
  - 支持分页查询
  - 多条件筛选：作者ID、关键词、分类ID、状态、时间范围
  - Redis缓存策略（3分钟+随机60秒过期）
  - 分布式锁防止缓存击穿

#### 2.2.3 文章编辑
- **需求描述**: 作者可修改自己的文章
- **可修改字段**: 标题、内容、分类
- **权限控制**: 只能修改本人文章
- **缓存策略**: 更新后清除相关缓存

#### 2.2.4 文章删除
- **需求描述**: 作者可删除自己的文章
- **权限控制**: 验证authorId必须匹配当前用户

#### 2.2.5 文章状态管理
- **改为草稿**: 将已发布文章改为草稿状态
- **发布文章**: 将草稿改为已发布状态
- **下架文章**: 将已发布文章下架

### 2.3 分类管理模块

#### 2.3.1 分类CRUD
- **添加分类**: 创建新的文章分类（1-50字符）
- **查询分类**: 支持分页查询分类列表和单个分类查询
- **更新分类**: 修改分类名称
- **删除分类**: 删除指定分类（需检查是否有文章关联）

### 2.4 评论模块

#### 2.4.1 发表评论
- **需求描述**: 用户对文章发表评论或回复其他评论
- **输入要求**:
  - 文章ID: 必填
  - 评论内容: 1-500字符
  - 父评论ID: 可选（回复评论时使用）
- **业务规则**:
  - 自动设置评论者ID
  - 支持树形评论结构
  - parentId为null表示一级评论

#### 2.4.2 查询评论
- **需求描述**: 获取文章的所有评论
- **返回格式**: 树形结构，包含所有子评论

### 2.5 社交互动模块

#### 2.5.1 点赞功能
- **点赞/取消点赞**: 用户对文章进行点赞或取消点赞
- **查询点赞用户列表**: 获取点赞某篇文章的所有用户
- **查询用户点赞列表**: 获取当前用户点赞的所有文章
- **业务规则**:
  - 点赞状态切换（status: 0未点赞/1已点赞）
  - 同步更新文章的点赞计数

#### 2.5.2 收藏功能
- **收藏/取消收藏**: 用户收藏或取消收藏文章
- **查询收藏列表**: 获取当前用户收藏的所有文章
- **业务规则**:
  - 收藏状态切换（status: 0未收藏/1已收藏）
  - 同步更新文章的收藏计数

#### 2.5.3 关注功能
- **关注用户**: 关注其他用户
- **取消关注**: 取消对某用户的关注
- **查询关注列表**: 获取用户关注的所有人
- **查询粉丝列表**: 获取用户的所有粉丝
- **查询关注状态**: 判断是否已关注某用户
- **关注动态**: 获取所有关注用户发布的文章列表

### 2.6 管理员模块

#### 2.6.1 管理员登录
- **需求描述**: 管理员通过独立入口登录
- **认证机制**: 与普通用户相同的JWT认证

#### 2.6.2 用户管理
- **查询用户列表**: 分页查询所有用户
- **用户搜索**: 支持按用户名模糊搜索
- **删除用户**: 管理员可删除任意用户

### 2.7 数据统计模块

#### 2.7.1 系统统计
- **统计指标**:
  - 文章总数
  - 用户总数
  - 分类总数
  - 最近7天文章数

#### 2.7.2 分类统计
- **需求描述**: 统计各分类下的文章数量
- **展示方式**: 支持图表展示

### 2.8 AI辅助写作模块

#### 2.8.1 内容润色
- **需求描述**: AI优化文章表达和语言风格
- **实现方式**: 流式响应（SSE）
- **超时控制**: 5分钟

#### 2.8.2 智能续写
- **需求描述**: 根据上下文继续创作内容
- **实现方式**: 流式响应（SSE）

#### 2.8.3 内容摘要
- **需求描述**: 自动生成文章摘要
- **实现方式**: 流式响应（SSE）

### 2.9 实时通知模块

#### 2.9.1 WebSocket推送
- **需求描述**: 当关注的作者发布新文章时实时通知粉丝
- **连接管理**:
  - 用户登录后建立WebSocket连接
  - 支持断线重连
  - 定时清理超时会话（5分钟无活动）
- **消息格式**:
  ```json
  {
    "type": "NEW_ARTICLE",
    "message": "您关注的博主发布了新文章",
    "fromId": "作者ID",
    "toId": "粉丝ID",
    "articleId": "文章ID",
    "timestamp": "时间戳"
  }
  ```

## 三、非功能需求

### 3.1 性能需求

#### 3.1.1 响应时间
- 文章列表查询: < 200ms（缓存命中）
- 单篇文章查询: < 100ms（缓存命中）
- 用户登录: < 500ms
- AI内容生成: < 5分钟

#### 3.1.2 并发能力
- 支持1000+并发用户在线
- 热点文章查询QPS > 1000
- WebSocket连接数 > 10000

#### 3.1.3 缓存策略
- **布隆过滤器**: 拦截99%的恶意请求
- **Redis缓存**:
  - 文章详情缓存: 5分钟
  - 文章列表缓存: 3分钟+随机60秒
  - RefreshToken: 7天
- **缓存命中率**: > 80%

### 3.2 安全需求

#### 3.2.1 认证授权
- JWT双Token机制（AccessToken + RefreshToken）
- Token黑名单机制
- 接口权限拦截器
- ThreadLocal用户信息透传

#### 3.2.2 数据安全
- 密码BCrypt加密存储
- 敏感词过滤（DFA算法）
- SQL注入防护（MyBatis-Plus）
- XSS防护

#### 3.2.3 权限控制
- 用户只能修改/删除自己的内容
- 管理员权限独立验证
- 接口级别的权限拦截

### 3.3 可用性需求

#### 3.3.1 容错机制
- Redis连接失败降级查询MySQL
- 分布式锁获取失败重试5次后降级
- WebSocket推送失败不影响主流程

#### 3.3.2 数据一致性
- Cache Aside模式（先更新DB后删除缓存）
- 分布式锁防止缓存击穿
- 随机过期时间防止缓存雪崩

#### 3.3.3 监控告警
- 布隆过滤器拦截率监控（正常>95%）
- 分布式锁竞争次数监控（异常<100次/秒）
- 缓存命中率监控（正常>80%）
- WebSocket连接数监控

### 3.4 可扩展性需求

#### 3.4.1 架构设计
- 多模块Maven项目（common、pojo、server）
- 分层架构（Controller-Service-Mapper）
- 统一异常处理
- 统一响应格式

#### 3.4.2 技术选型
- Spring Boot 2.6.13（易于升级）
- MyBatis-Plus（支持多数据源）
- Redis（支持集群模式）
- RabbitMQ（支持消息重试）

### 3.5 可维护性需求

#### 3.5.1 日志记录
- AOP统一日志记录（@SystemLog注解）
- 记录内容：操作模块、业务类型、请求参数、执行时间、异常信息
- 日志存储：sys_oper_log表

#### 3.5.2 代码规范
- RESTful API设计
- 参数校验（JSR-303）
- 代码注释规范
- 统一命名规范

## 四、技术架构

### 4.1 技术栈

#### 4.1.1 后端技术
- **编程语言**: Java 17
- **Web框架**: Spring Boot 2.6.13
- **ORM框架**: MyBatis-Plus 3.5.3.1

- **安全框架**: Spring Security
- **认证方式**: JWT (jjwt 0.11.5)

#### 4.1.2 前端技术
- **框架**: React 19.2.0
- **路由**: React Router DOM 7.13.0
- **HTTP客户端**: Axios 1.13.4
- **构建工具**: Vite 5.4.11
- **语言**: TypeScript 5.9.3

### 4.2 系统架构

#### 4.2.1 分层架构
```
表现层 (Controller)
    ↓
业务逻辑层 (Service)
    ↓
数据访问层 (Mapper)
    ↓
数据库 (MySQL)
```

#### 4.2.2 缓存架构
```
请求 → 布隆过滤器 → Redis缓存 → 分布式锁 → MySQL
```

#### 4.2.3 认证架构
```
用户请求 → 拦截器 → JWT验证 → ThreadLocal → 业务处理
```

### 4.3 数据库设计

#### 4.3.1 核心表结构
- **admin**: 管理员表（id, username, password, token）
- **author**: 用户表（id, username, password）
- **article**: 文章表（id, category_id, title, content, author_id, create_time, update_time, status, thumbs_up, favorites）
- **category**: 分类表（id, name）
- **comment**: 评论表（id, article_id, author_id, parent_id, content, create_time）
- **favorites**: 收藏表（id, author_id, article_id, favorites_time, status）
- **follows**: 关注表（id, author_id, target_id, follow_time, status）
- **thumbs_up**: 点赞表（id, author_id, article_id, thumbs_up_time, status）
- **sys_oper_log**: 系统操作日志表

#### 4.3.2 索引设计
- article表: idx_category（category_id）
- favorites表: idx_author_id, idx_article_id
- follows表: uk_author_target（author_id, target_id）, idx_author_id, idx_target_id
- thumbs_up表: idx_author_id, idx_article_id

## 八、风险评估

### 8.1 技术风险
- **Redis单点故障**: 建议使用Redis集群或哨兵模式
- **MySQL性能瓶颈**: 建议配置主从复制和读写分离
- **WebSocket连接数限制**: 建议使用消息队列解耦

### 8.2 安全风险
- **JWT令牌泄露**: 使用HTTPS传输，设置合理的过期时间
- **敏感词库维护**: 定期更新敏感词库
- **DDoS攻击**: 建议使用CDN和限流策略

### 8.3 业务风险
- **内容审核**: 需要人工审核机制配合AI过滤
- **用户隐私**: 遵守数据保护法规
- **版权问题**: 明确用户协议和版权声明

## 九、总结

本博客系统是一个功能完善、架构合理的现代化内容平台，具备以下核心优势：

1. **高性能**: 通过多层缓存和分布式锁机制，实现高并发场景下的稳定运行
2. **高安全**: JWT双Token认证、BCrypt加密、敏感词过滤等多重安全保障
3. **高可用**: 降级策略、容错机制、监控告警确保系统稳定性
4. **易扩展**: 模块化设计、分层架构、统一规范便于后续功能扩展
5. **智能化**: 集成AI辅助写作，提升用户创作体验

系统适用于中小型内容平台、社交媒体、知识社区等高并发读多写少的业务场景。

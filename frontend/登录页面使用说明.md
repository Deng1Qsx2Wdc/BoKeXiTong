# 登录页面使用说明

## 功能概述

登录页面已经开发完成，支持以下功能：

### 1. 双重登录模式
- **用户登录**：普通用户登录入口
- **管理员登录**：管理员登录入口
- 可通过页面顶部的切换按钮在两种模式间切换

### 2. 表单验证
- **用户名验证**：
  - 不能为空
  - 长度必须在 3-20 个字符之间
- **密码验证**：
  - 不能为空
  - 长度必须在 6-20 个字符之间
- 实时显示错误提示信息

### 3. 登录功能
- 调用后端 API 进行身份验证
- 用户登录：`POST /author/login`
- 管理员登录：`POST /admin/login`
- 登录成功后：
  - 将 JWT Token 存储到 localStorage
  - 将用户信息存储到 localStorage
  - 自动跳转到相应页面（用户跳转到首页，管理员跳转到仪表盘）

### 4. 用户体验优化
- 加载状态显示（登录按钮显示加载动画）
- 错误提示（显示详细的错误信息）
- 响应式设计（适配不同屏幕尺寸）
- 美观的渐变背景和卡片式布局

## 技术实现

### 使用的技术栈
- **React 18**：前端框架
- **TypeScript**：类型安全
- **React Router**：路由管理
- **Axios**：HTTP 请求
- **Tailwind CSS**：样式框架

### 文件结构
```
frontend/src/
├── pages/
│   └── Login.tsx          # 登录页面组件
├── api/
│   ├── client.ts          # Axios 客户端配置
│   ├── author.ts          # 用户 API
│   └── admin.ts           # 管理员 API
├── types/
│   └── index.ts           # TypeScript 类型定义
├── App.tsx                # 应用主组件（路由配置）
└── main.tsx               # 应用入口

```

### API 集成
- 使用 Axios 拦截器自动添加 JWT Token
- 统一的错误处理
- 响应拦截器处理业务错误和 HTTP 错误

## 如何运行

### 1. 启动后端服务
确保后端服务已经启动在 `http://localhost:8080`

### 2. 启动前端开发服务器
```bash
cd frontend
npm run dev
```

### 3. 访问登录页面
打开浏览器访问：`http://localhost:5173/login`

## 注意事项

1. **环境变量配置**：
   - API 基础 URL 配置在 `.env` 文件中
   - 默认值：`VITE_API_BASE_URL=http://localhost:8080`

2. **跨域问题**：
   - Vite 已配置代理，将 `/api` 请求转发到后端
   - 如果遇到跨域问题，检查 `vite.config.ts` 中的代理配置

3. **Token 管理**：
   - Token 存储在 localStorage 中
   - 请求拦截器会自动添加 `Authorization: Bearer <token>` 头
   - 401 错误会自动清除 Token 并提示重新登录

4. **类型安全**：
   - 所有 API 请求和响应都有完整的 TypeScript 类型定义
   - 类型定义位于 `src/types/index.ts`
